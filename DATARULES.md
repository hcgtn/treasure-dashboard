# 预警引擎的四条规则
    每个工厂遍历这四条规则，命中即生成一条预警：
    规则1：库存总量预警
    total > 200kg  → 🔴 红色（超警戒线）
    total > 150kg  → 🟡 黄色（接近警戒线）
    规则2：信用评分预警
    score < 60     → 🔴 红色（信用过低）
    score < 75     → 🟡 黄色（信用偏低）
    规则3：单品类占比失衡
    任一品类/总量 > 60% → 🟡 黄色（结构风险）
    规则4：品类缺失预警
    任一品类 = 0kg → 🟡 黄色（品类缺失）

| 你想生成… | 就把工厂数据改成… |
|-----------|-------------------|
| 🔴 红色 | 总量 > 200kg（超警戒线） |
| 🔴 红色 | 信用评分 < 60（信用过低） |
| 🟡 黄色 | 总量 150~200kg（接近警戒线） |
| 🟡 黄色 | 信用评分 60~75（信用偏低） |
| 🟡 黄色 | 任一品类/总量 > 60%（结构失衡） |
| 🟡 黄色 | 任一品类 = 0kg（品类缺失） |


| # | 规则 | 判定条件 | 级别 |
|---|------|----------|------|
| R1 | 库存总量 | total > 200 → 🔴 / total > 150 → 🟡 | 互斥（二选一） |
| R2 | 信用评分 | score < 60 → 🔴 / score < 75 → 🟡 | 互斥（二选一） |
| R3 | 单品类占比 | 金水/金块/成品任一占比 > 60% → 🟡 | 每品类独立判断 |
| R4 | 品类缺失 | 金水/金块/成品任一类 = 0 → 🟡 | 每品类独立判断 |
